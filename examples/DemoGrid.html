  • Responsive sizes only apply when the screen is their size. If there are no smaller responsive sizes, then the collumn will stack.
  • If there are no larger responsive sizes, then the that responsive size is used for all larger sizes.
  • Responsive sizes are used from the most specific to least. In this order. xl -> lg -> md -> sm -> col.
  • col is the the dynamic size. Its both the smallest and the largest because it can be any size depending on the other columns. Col will never automatically stack.
  • col is the same as xs. You never explicitly declare xs, its always implied by col, or its absence.
